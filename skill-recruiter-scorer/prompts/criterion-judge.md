# Промпти агента-рекрутера (для довідки)

Скрипт `score_candidates.py` містить ці промпти всередині. Тут вони окремо - щоб розуміти логіку і за потреби підкрутити під себе.

---

## 1. Вакансія → критерії з вагами (один раз на вакансію)

```
Ти технічний рекрутер. Виведи 4-6 критеріїв найму з вагами.

Вакансія: {job}

Поверни СТРОГО JSON-масив:
[{"criterion":"назва","weight":0.30,"must_have":true,"min_years":5,"keywords":["python","go"]}]

Правила:
- weight 0-1, сума = 1.0. Must-have - найбільша вага.
- must_have: true якщо обовʼязкове (відсів).
- min_years: число років для критерію про досвід, інакше 0.
- keywords: 2-5 слів-маркерів (для перевірки в коді).
```

---

## 2. Оцінка кандидата по ОДНОМУ критерію (на кожен критерій кожного кандидата)

```
Ти технічний рекрутер. Оціни кандидата ТІЛЬКИ по ОДНОМУ критерію.

Критерій: {criterion} (must-have: {must_have})
Профіль: {profile}

Поверни СТРОГО JSON:
{"score":7,"evidence_quote":"дослівна цитата з профілю","reasoning":"1 рядок"}

Правила:
- score 0-10 на основі ТІЛЬКИ профілю.
- evidence_quote: дослівний фрагмент. Немає доказів → score 0, evidence_quote "ДОКАЗІВ НЕ ЗНАЙДЕНО".
- НІКОЛИ не вигадуй досвід, якого немає.
- Читай смисл: "розробляв на Пайтоні" = Python-досвід.
```

---

## Що тут робить КОД, а не LLM (суть гібриду)

| Що | Хто рахує | Чому |
|---|---|---|
| Роки досвіду | **КОД** (з дат роботи) | LLM бреше в числах |
| Бал по критерію «X років» | **КОД** перетирає бал LLM | детерміновано: маєш роки чи ні |
| Must-have відсів (knockout) | **КОД** (hard-фільтр) | обовʼязкове - не «на думку моделі» |
| Зважена сума → fit_score | **КОД** | прозоро, перевірювано, аудит-слід |
| Суддення «підходить за смислом» | **LLM** | тільки модель читає живий текст |
| Цитата-доказ | **LLM** | витягує підтвердження з профілю |

> Інженерне правило уроку: **числам - код, судженню - модель.** Чистий LLM наврав би в роках; чистий код відсіяв би «Пайтон» без англ. Гібрид бере найкраще від обох.
