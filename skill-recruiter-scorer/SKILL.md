# Recruiter Scorer - оцінка кандидатів під вакансію

Скіл бере вакансію + список лідів (LinkedIn-URL), тягне профіль кожного через HarvestAPI, **сам** оцінює кожного по 5 критеріях і видає готову таблицю прямо в чаті.

**Що на виході (в чаті):** таблиця, де по кожному кандидату - **бал** (загальна сума), **критерії оцінки**, **чому саме такий бал** (доказ із профілю), і **готове повідомлення**, яке варто йому відправити.

---

## Потік (4 кроки, інтерактивно)

### Крок 1 - Попроси вакансію
Скажи юзеру:
> «Дай вакансію: встав текст або шлях до файлу (.md/.txt/Google Drive експорт).»

Прийми будь-який варіант. Якщо опис мутний/короткий - попроси додати **обовʼязкові вимоги (must-have)**, бо без них критерії будуть слабкі.

### Крок 2 - Попроси список лідів
Скажи:
> «Дай список лідів: LinkedIn-посилання кандидатів, по одному в рядок (або шлях до .txt / .csv зі стовпцем посилань).»

Збережи посилання у файл `leads.txt` робочої папки (по одному в рядок). Прийми і повні URL, і слаги.

### Крок 3 - Попроси ключ HarvestAPI (якщо доступу ще немає)
Перевір змінну `APIFY_TOKEN`:
```bash
[ -n "$APIFY_TOKEN" ] && echo "ключ є" || echo "ключа немає"
```
Якщо ключа немає - скажи:
> «Потрібен доступ до HarvestAPI. Дай свій Apify API-ключ (HarvestAPI actor) - я підставлю його як `APIFY_TOKEN`. Взяти ключ: apify.com → Settings → Integrations → API token.»

Дочекайся ключа. Не вигадуй, не йди далі без ключа (інакше профілі не дістати).

### Крок 4 - Тягни профілі + оціни + видай таблицю
1. **Фетч профілів** одним endpoint:
   ```bash
   APIFY_TOKEN="<ключ>" bash fetch-profiles.sh leads.txt _raw/
   ```
   Скрипт по кожному посиланню дьоргає Apify actor `harvestapi/linkedin-profile` → кладе профіль у `_raw/cand-N.json`. Один endpoint, нічого зайвого. Якщо профіль не дістався - файл містить `{"_fetch_failed":true}`, кандидат піде в окрему секцію, не зникне молча.

2. **Оціни кожного САМ** (ти і є модель - рахуй у голові, не зови зовнішній LLM):
   - Прочитай `_raw/*.json` (кожен профіль) + вакансію.
   - **Виведи РІВНО 5 критеріїв** з вакансії (must-have - найважливіші). Признач кожному вагу (сума ваг = 100%).
   - По кожному критерію постав бал **0-10** на основі ТІЛЬКИ профілю + **дослівну цитату-доказ** із профілю. Немає доказу → 0 + «доказів не знайдено» (не вигадуй).
   - Порахуй **загальну суму**: `сума(вага × бал) × 10` → 0-100.
   - **Читай смисл, не ключові слова:** «розробляв на Пайтоні» = Python-досвід.
   - Must-have не закритий → жорстко занижуй загальний бал (це відсів, а не «майже підходить»).

3. **Видай таблицю прямо в чаті** (формат нижче). Не пиши у файли, не в Google Sheets - просто покажи в цьому вікні.

## Формат виходу (в чаті)

### 1) Зведена таблиця - ранжована за балом
```
| # | Кандидат | Бал /100 | Вердикт | 5 критеріїв (бал) |
|---|----------|----------|---------|-------------------|
| 1 | Імʼя     | 88       | ЗАПРОСИТИ | Python 9 · Досвід 10 · API 9 · SQL 8 · Англ 6 |
| 2 | Імʼя     | 41       | ВІДСІВ    | ... |
```
Вердикт: **ЗАПРОСИТИ** (бал ≥ 70 і must-have закриті) / **МОЖЛИВО** (50-69, дивиться людина) / **ВІДСІВ** (< 50 або провалив must-have).

### 2) По кожному кандидату - деталі
```
## Імʼя кандидата - 88/100 · ЗАПРОСИТИ

Критерії оцінки:
| Критерій | Вага | Бал | Чому такий бал (доказ із профілю) |
|----------|------|-----|----------------------------------|
| Python 5+ р. (must) | 30% | 9 | "7 років комерційної розробки на Python" |
| Досвід backend     | 25% | 10 | "Senior Backend @ FinTech, 3 роки" |
| ... (ще 3 критерії) | | | |

Повідомлення кандидату:
> Вітаю! Помітив ваш досвід із [сильна сторона]. У нас відкрита роль [назва] - 
> схоже на те, що ви вже робите. Цікаво обговорити 15 хв?
```
Повідомлення: до 90 слів, під конкретну сильну сторону з доказів, без тире, без емодзі.

### 3) Окремо
- **МОЖЛИВО** (сіра зона) - список, людина вирішує.
- **Не дістали профіль** (`_fetch_failed`) - список посилань, які не відкрились.

### 4) Дисклеймер (обовʼязково, останній рядок)
> **AI оцінив - рішення за вами.** Не для авто-відмови без людини (EU AI Act, GDPR ст.22). «МОЖЛИВО» обовʼязково дивиться людина.

---

## Hard rules

1. **5 критеріїв на кандидата.** Не більше, не менше - рівно 5, виведених з вакансії, з вагами (сума = 100%).
2. **Доказ-цитата на кожен бал.** Немає цитати з профілю → бал 0. Захист від вигадування + слід «чому такий бал».
3. **Читай смисл, не ключові слова.** «Пайтон» = Python. Це і є перевага агента над keyword-скриптом.
4. **Must-have - відсів, не вага.** Кандидат без обовʼязкового → ВІДСІВ, а не «майже».
5. **Один endpoint.** Тільки `fetch-profiles.sh` (HarvestAPI / Apify actor harvestapi/linkedin-profile). Нічого більше не дьоргати.
6. **Профіль тільки з фетчу.** Не вигадувати досвід, якого немає в JSON. `_fetch_failed` → окрема секція, не мовчазний пропуск.
7. **Людина в контурі.** Дисклеймер обовʼязковий.
8. **M-dash заборонений** ніде у виводі (тільки дефіс із пробілами).

## Sample-дані (для тренування без реальних лідів)

`sample-data/` містить готові профілі у форматі HarvestAPI + вакансію:
- `job-backend.md` - вакансія Senior Backend
- `candidate-1.json` (сильний) · `candidate-2.json` (слабкий, frontend) - щоб перевірити, що скіл РАНЖУЄ

Прогін на sample (без ключа, профілі вже є):
```bash
# профілі вже в sample-data/ - пропусти фетч, читай їх напряму і оціни
```
Прочитай `sample-data/job-backend.md` + обидва `sample-data/candidate-*.json` → оціни по 5 критеріях → видай таблицю. Очікувано: сильний ~85+ ЗАПРОСИТИ, frontend ~40 ВІДСІВ по must-have.
