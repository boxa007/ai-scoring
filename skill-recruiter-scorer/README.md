# Агент-рекрутер: оцінка кандидатів під вакансію

Скіл для Claude Code. Даєш вакансію + список лідів (LinkedIn-посилання) → скіл тягне профіль кожного через HarvestAPI → сам оцінює по 5 критеріях → видає готову таблицю прямо в чаті: бал, критерії, чому такий бал, готове повідомлення кандидату.

---

## Встановлення

Скопіюй папку `skill-recruiter-scorer/` у свій проєкт як скіл:
```
<твій-проєкт>/.claude/skills/recruiter-scorer/
├── SKILL.md
├── fetch-profiles.sh
├── README.md
└── sample-data/
```
Claude Code підхопить скіл за описом. Запуск: напиши `/recruiter-scorer` або «оціни кандидатів».

---

## Що тобі треба

1. **Вакансія** - текст або файл (.md/.txt). Можна експорт із Google Drive.
2. **Список лідів** - LinkedIn-посилання кандидатів, по одному в рядок (.txt або .csv).
3. **Apify API-ключ** (HarvestAPI) - apify.com → Settings → Integrations → API token.
   ```bash
   export APIFY_TOKEN=apify_api_xxxxxxxx
   ```

---

## Потік (що робить скіл)

```
ти кажеш "/recruiter-scorer"
  → скіл: "дай вакансію"           вставляєш / шлях до файлу
  → скіл: "дай список лідів"        LinkedIn-посилання, по одному в рядок
  → скіл: "дай Apify-ключ"          якщо APIFY_TOKEN ще не виставлений
  → fetch-profiles.sh по кожному посиланню дьоргає HarvestAPI → профіль у _raw/
  → скіл САМ оцінює кожного: 5 критеріїв × бал + цитата-доказ → загальна сума
  → готова таблиця ПРЯМО В ЧАТІ: бал · критерії · чому · повідомлення
```

Один endpoint (HarvestAPI actor `harvestapi/linkedin-profile`), нічого зайвого. Рахує сам Claude - окремий OpenAI-ключ НЕ потрібен.

---

## Пробний запуск (без реальних лідів)

У `sample-data/` уже є вакансія + 2 профілі (сильний і слабкий). Скажи скілу:
> «Оціни кандидатів із sample-data під вакансію sample-data/job-backend.md»

Скіл прочитає профілі напряму (фетч не потрібен - вони вже є), оцінить, видасть таблицю. Очікувано: сильний backend ~85+ ЗАПРОСИТИ, frontend ~40 ВІДСІВ по must-have. Так перевіриш, що скіл працює, ще до підключення ключа.

---

## Формат виходу (в чаті)

1. **Зведена таблиця** - ранжована за балом: `# · Кандидат · Бал/100 · Вердикт · 5 критеріїв`
2. **По кожному кандидату:** таблиця 5 критеріїв (вага · бал · доказ-цитата) + готове повідомлення (до 90 слів)
3. **Окремо:** МОЖЛИВО (сіра зона) + не дістали профіль
4. **Дисклеймер:** AI оцінив, рішення за людиною

---

## ⚠️ Етика (hard, з уроку)

AI-скоринг кандидатів - юридично high-risk (EU AI Act, GDPR ст.22). Скіл завжди друкує:
> **AI оцінив - рішення за вами.** Не для авто-відмови без людини. «МОЖЛИВО» дивиться людина.

Агент дає РЕКОМЕНДАЦІЮ, не рішення. Не використовуй для авто-відмови.
